实习第四天

今天主要完成了项目两个页面和另一个板块的搜索功能，需求是在页面一输入关键字搜索后跳转到页面二，显示出搜索结果，如果用户在页面一没有输入有效内容(比如就打了几个空格)，那么点击搜索后进入页面二会有个历史记录列表，点击右侧X可删。也就是说输入了有效关键字，显示搜索结果，反之则显示搜索记录。

具体需求对应算法

  • 首先判断用户输入的内容是否合法，我就用了一个正则表达式去掉空格，判断用户输入的字符串在去掉空格后的长度是否大于等于1？

  • 既然是存在localStorage中那么页面v-for的渲染肯定是以localStorage.getItem(..)为遍历对象，那么就有新的问题了，如果用户是第一次搜索，当前本地缓存内没有值呢？既然没有值，那么去取的时候就会是null，下面是我的解决办法

为什么要以数组形式存入呢？因为对象行驶的话push、unshift等方法是用不了的，也就是说后面有新的历史纪录后就不好操作了。另外说一下以JSON.stringify()形式存入的原因，因为如果直接存入数组对象的话，你取出直接用localStorage.getItem(xxx)给你的结果是”[object Object],[object Object],[object Object]”这种，显然是没用的，用JSON.parse方法会直接报错uncaught syntaxerror,我直翻一下大概是未捕获的语法错误，所以我用了JSON.stringify()。

  • 如果假设用户输入关键字合法，但是有个关键字输入了四五遍，那么总不能重复存入四五遍该历史记录吧？我的解决方式是将输入的关键字与当前localStorage中的所有历史关键字比较，使用了map方法，不过我觉得我这种方法只适合单个数据的去重

感觉真的不难，就是自己基础不太扎实，用不出简单的办法，老是声明很多其他的数组，最后写的很复杂。另外还有删除历史记录，也不难，拿到用户点击的删除的那条记录的索引就能完成后续操作了，毕竟是以unshift推入的，因此历史记录列表的索引与在本地缓存中的索引是一样的。
